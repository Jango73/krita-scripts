ComfyUI Image Enhance

Copyright 2025-2026 Jango73 - github.com/Jango73

This plugin runs ComfyUI workflows directly from Krita to enhance the full image and optional region selections.

Show the ComfyUI Workflow docker

  1. Open Krita menu Settings > Dockers.
  2. Enable ComfyUI Workflow. The docker appears in the right panel (tool options area); you can drag it if needed.

Usage

  1. Open Tools > Scripts > ComfyUI Image Enhance.
  2. On the Server & Workflows tab, set the ComfyUI server URL, workflows folder, and the global/region workflow file names.
  3. On the Workflow tab, fill the global prompt and up to four region prompts (left to right order for regions), and add parameter names (e.g., CFG.value) and values for global and region runs. The “Copy global to region” button duplicates the list.
  4. Optionally select one or more rectangular areas; each becomes a region pass. Without a selection only the global pass runs.
  5. Click Go (global + regions) or Regions (regions only). The Log tab shows all settings, execution, and errors.

Notes

  • The prompt node in the workflow must be named Prompt. The image input node is detected automatically.
  • Results are added as new layers using the configured opacity; region layers inherit the selection position with a soft edge fade.
  • The output image can be deleted after import if you enable the setting; this is a permanent delete, not a move to the trash.
  • Use the Copy log button to put the full log into the clipboard for debugging.
  • Placeholders: you can reference {width}, {height}, and {best-scale} (computed as clip(640/max(width,height), 0.2..1.0)) inside prompts or parameter values; they are resolved per input image (global or region) before sending to ComfyUI.
  • Custom nodes required by Universal.json: ImpactConditionalBranch, ImpactImageInfo, ImpactCompare, ImpactLogicalOperators (from ComfyUI Impact Pack), BNK_TiledKSampler (from ComfyUI_TiledKSampler), and DZ_Face_Detailer (Face Detailer add-on). Install these extensions in ComfyUI to avoid “unknown node” errors.

How Universal.json works (default enhance workflow)

The pipeline runs several sampling stages (samplers), with resolution changes.

Input size mode

Controls how the input image resolution is prepared before the first sampler.

  • 0 : Original
    The input image keeps its original resolution.
  • 1 : Fit latent
    The input image is resized to match the pipeline working resolution (latent size), which would be 512x512.
  • 2 : Scale factor
    The input image is resized using the value defined in Scale input amount.

Scale input amount

Defines the scale factor used when Input size mode = Scale factor.

Input size mode = Scale factor
Scale input amount = 0.5

The input image will be reduced to 50% of its original size before the first sampler.

This parameter has no effect unless the Scale factor mode is selected.

Initial stage

This stage generates the base structure of the image. No scaling occurs during the sampler itself. It is meant to be used only in image creation, not enhancement.

Steps / Refine steps

Controls diffusion steps during sampling.

CFG / Refine CFG

Controls prompt guidance strength during sampling.

Lower values preserve the structure generated by the initial stage. Higher values push the refinement stage to follow the prompt more strongly.

This parameter does not affect resolution.

Denoise / Refine denoise

Controls how strongly the refinement samplers modify the image.

Higher values allow larger changes during the refinement stage.

This parameter does not affect resolution.

Face detailer

Enables an additional face refinement sampler applied near the end of the pipeline.

This stage operates on the current image resolution and does not introduce a new scale change.
